05. Featured Case Study

PHASE 1: OFFENSE (RED TEAM)

SMB Brute Force Execution

Using Kali Linux, I simulated an unauthorized access attempt targeting the Windows 7 endpoint. I utilized Hydra to execute a dictionary attack against the SMB protocol (Port 445), deliberately generating high-volume authentication traffic to test the SIEM's threshold settings.

CMD: hydra -l Manu -p wrongpass smb://192.168.20.7
PHASE 2: DETECTION (BLUE TEAM)

Real-Time Alert Triggering

The Wazuh Manager successfully ingested logs from the Windows Agent. The custom Audit Policy I configured (Event ID 4625) ensured failures were recorded. The dashboard immediately visualized a spike in severity, triggering Rule ID 60122 (Logon Failure) 11 times in rapid succession.

Wazuh SIEM detecting the spike in failed login attempts in real-time.
PHASE 3: FORENSICS & ANALYSIS

IOC Identification

Drilling down into the alert data provided concrete Indicators of Compromise (IOCs). I successfully correlated the timestamp of the alerts with the attacker's machine. The log data confirmed the Source IP 192.168.20.8 and the targeted user account "Manu", validating the nature of the attack.

Forensic analysis confirming the attack source originated from 192.168.20.8
